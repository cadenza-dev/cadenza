# Speaker Notes

Speaker notes should help a presenter deliver the talk without changing the
visible timeline.

## Do

- Add notes with `Notes`.
- Keep notes concise unless the user asks for a full script.
- Name the audience takeaway before adding delivery nuance.
- Place notes close to the slide or reveal they support.
- Preserve presenter metadata: current slide, current step, notes, elapsed wall
  time, and active presenting time.

## Avoid

- Duplicating every visible bullet.
- Hiding presenter-only text in visual layout.
- Assuming step-level notes exist unless the current implementation supports the
  needed association. Prefer slide-level notes when uncertain.

## Good Notes

Good notes sound like prompts for a speaker:

- "Pause here and connect the compiler to the demo."
- "Mention that export support is not claimed in this phase."
- "Ask the audience to notice the diagnostic source ID."
