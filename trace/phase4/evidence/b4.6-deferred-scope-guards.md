# B4.6 Deferred-Scope Guards

Batch: `B4.6`
Scenarios: `TC-STAR-003`
Requirements: `DOGF-005`, `TYPO-005`, `TRPR-005`, `STAR-005`, `STAR-006`

## Read-Only MCP Disposition

Read-only MCP is not implemented in Phase 4. It remains a closeout or
Phase 5-start evaluation candidate, consistent with
`wip/future-support/phase-5-candidates.md`.

## Tool-Based MCP Absence

Tool-based MCP remains absent. The B4.6 guard checks that no Phase 4 artifact
creates MCP package or command surfaces for `validate_deck`, `render_preview`,
or `inspect_composition`.

## Starter Boundary

Phase 4 starter work stays in `cadenza-best-practices` guidance plus TSX
examples. The starter artifacts reject generic prompt-to-deck use, broad
template catalogs, direct visual editors, distribution/runtime promises, stable
public API promises, outside-user validation claims, and MCP implementation
claims.
