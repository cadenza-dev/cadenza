## Finding

`VARR-B4.3-001` records a diagnostics finding on
`product-layer-loop` in the `product-layer` chapter. The authored talk mentioned
traceable repair evidence, but the local workflow did not make visual acceptance
sign-off or waiver routing visible enough for maintainer review.

`VARR-P4-CLOSEOUT-001` records a maintainer-observed presenter workflow finding
from closeout visual review. The right-side presenter panel advanced through
semantic slides and steps, but the left Remotion Player visual output remained
static, exposed presenter-only notes and a font readiness marker, and showed a
stale `Goto render-safe-demo` control that does not target a Phase 4 dogfood
slide.

`VARR-P4-CLOSEOUT-002` records a maintainer-observed typography/layout finding
from closeout visual review. On `preview-reliability-budget` step 2, the
`Compiler evidence...` dense copy was visibly clipped near the bottom of the
left Remotion Player. The objective cause was mixed: the authored sample used a
too-small compact `TypographyBox`, and the preview renderer measured auto-fit
before applying the configured base typography style while allowing the
preceding `MediaFrame` to take too much vertical space.

## Repair

The repair stays outside framework internals:

- `examples/phase4/dogfood-talk.tsx` now names visual acceptance evidence and
  maintainer sign-off or waiver in the authored deck.
- `examples/phase4/preview.ts` exposes a product-layer visual acceptance
  diagnostic.
- `examples/phase4/preview.jsx` renders that diagnostic in the same-browser
  presenter panel.

No `packages/**/src/**` framework edit is recorded as the dogfood-talk repair
surface.

The closeout visual-navigation repair is intentionally routed as a framework
defect, not as normal dogfood-talk authored repair:

- `packages/preview-remotion/src/CadenzaPlayer.tsx` now renders the Remotion
  composition from the current frame, so the visual slide follows semantic
  slide navigation and cumulative step reveal.
- `packages/preview-remotion/src/CadenzaPlayer.tsx` keeps render-safe resource
  readiness evidence in a hidden preload layer instead of rendering all future
  slide content visibly.
- `packages/preview-remotion/src/CadenzaPlayer.tsx` hides presenter-only notes
  in the Player visual surface and removes the stale `Goto render-safe-demo`
  button.
- `packages/preview-remotion/src/render-safe/SafeFontPreview.tsx` keeps font
  readiness diagnostics observable while preventing font markers from appearing
  in the visual slide.
- `tests/browser/remotion-preview.spec.ts` verifies the Phase 4 dogfood preview
  shows the current slide, reveals cumulative step content, hides notes and font
  markers, removes the stale button, and switches to `timeline-compiler` after
  navigation.

The dense-copy clipping repair is also routed as a mixed authored-deck plus
framework-defect repair:

- `examples/phase4/dogfood-talk.tsx` now shares
  `PHASE4_RELIABILITY_DENSITY_BOX` for the reliability-budget copy and gives the
  compact text enough room to fit while preserving the density-budget pressure.
- `examples/phase4/preview.ts` uses the same shared box dimensions for the
  presenter typography-density diagnostic, avoiding trace drift between the
  authored slide and the diagnostic surface.
- `packages/preview-remotion/src/CadenzaPlayer.tsx` measures
  `TypographyBoxPreview` after applying base auto-fit typography, applies the
  fitted result to the live DOM, constrains slide layout, and caps
  `MediaFramePreview` width by aspect-ratio-derived height.
- `tests/browser/remotion-preview.spec.ts` verifies the
  `preview-reliability-budget-density` box reaches `fitted`, has no live scroll
  overflow, and remains above the Player controls guard.

The pixel-sanity note is supplemental reviewer evidence only. It does not
replace structured findings, local preview diagnostics, maintainer sign-off, or
an explicit maintainer waiver.
